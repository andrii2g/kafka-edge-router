# Security policy

## Supported versions

Until the first release, only the current `main` branch receives security fixes. After a
stable release, this table will list supported minor versions.

## Reporting a vulnerability

Do not open a public issue for a suspected vulnerability. Use GitHub private vulnerability
reporting for this repository, or contact the repository owner through a private channel
listed in the GitHub profile.

Include:

- affected commit or version;
- threat scenario and prerequisites;
- reproducible steps or proof of concept;
- tenant, credential, SSRF, availability, or integrity impact; and
- suggested mitigation when available.

Remove real credentials and customer data. You should receive acknowledgement according
to the repository owner's published response policy once the repository is created.

See `docs/SECURITY.md` for the technical threat model and known gaps.
